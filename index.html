01 / 增长路径总览

先把前端获客、私域承接、咨询转化和售后裂变串成一条线。

世理法源的增长链路不只看单个账号或单个社群，而是从用户第一次看到法律内容开始，一直承接到咨询、成交、服务、转介绍和内容反哺。

01

前端入口负责持续进线

通过 IP 起号、小红书图文矩阵、截流和电商搜索渠道，承接不同场景下有法律需求的用户。

02

私域承接负责分层培育

用户进入企微或个人微信后，通过欢迎语、资料承接、需求标签、1v1 跟进和社群资料内容完成持续培育。

03

后端转化负责形成闭环

咨询成交后继续做好售后反馈、案例沉淀、复购升单和转介绍，再把真实问题反哺到前端内容渠道。

01

公域获客入口

通过多渠道获取有法律需求或潜在法律需求的用户，前端不急着成交，先把用户从公域内容里筛出来。

主要渠道

  • IP 起号：律师/法律主理人短视频、口播、案例解析
  • 小红书图文矩阵：婚姻、债务、合同、劳动、房产、企业法务等场景图文
  • 截流打法：同行笔记评论区、自营打窝笔记、关键词截流
  • 电商搜索渠道：法律资料包、合同模板、维权流程、证据整理表

对应交付

IP 定位对标拆解选题库内容脚本图文矩阵 SOP截流话术资料品设计引流路径设计
02

兴趣用户触达

用户通过内容、搜索、资料、评论区互动产生兴趣，开始与世理法源发生第一次连接。

用户行为

  • 看完视频后私信或评论
  • 看小红书图文后咨询
  • 在评论区被内容吸引
  • 搜索法律资料后下单
  • 通过资料钩子进入私域

业务动作

评论区承接私信关键词回复资料钩子设计低门槛资料包安全导粉路径前端数据记录
03

私域承接分层

用户进入企微/个人微信后，先完成欢迎、需求判断、标签分层。法律行业不能上来硬推服务，先判断问题和紧急程度。

承接动作

  • 自动欢迎语与人工接待话术
  • 用户问题收集与法律需求标签
  • 紧急程度判断和用户价值分层
  • 朋友圈、1v1、社群承接路径衔接

分层建议

资料型用户咨询型用户急迫型用户高价值用户企业客户
04

用户培育转化

通过朋友圈、1v1 和法律资料社群持续培育，让用户从“我有点问题”进入“我愿意咨询/愿意付费”。

培育内容

  • 法律问题科普、常见误区提醒
  • 真实案例拆解、咨询前准备清单
  • 资料使用说明、不同问题处理路径
  • 律师服务价值说明、风险提示

对应交付

用户成长旅程 SOP1v1 培育话术朋友圈 SOP社群运营 SOP咨询转化话术活动机制
05

咨询成交与服务

根据用户需求程度，设计从低门槛资料到付费咨询，再到专项律师服务/企业法务顾问的转化路径。

资料包 / 模板资料 → 低门槛咨询 / 电话咨询 → 律师初步方案判断 → 案件委托 / 专项服务 / 企业法务顾问

售后动作

  • 资料发送与使用说明
  • 咨询后结论整理和下一步建议
  • 案件节点同步和风险提醒

03 / 四大前端获客打法

不同渠道承担不同任务，不是所有入口都负责直接成交。

IP 建信任，小红书接搜索，截流抢高意向，电商搜索用低门槛资料完成首单激活。四个入口要互相配合。

IP

IP 起号

定位：建立专业信任和律师 IP 认知。

内容方向：案例解析、法律误区、维权流程、老板法务提醒、婚姻/债务/合同/劳动等场景内容。

目标：让用户相信“这个账号懂我的问题，可以先咨询”。

IP 定位选题库脚本拍摄指导视频剪辑数据复盘
图

小红书图文矩阵

定位：承接搜索型用户。

内容方向：法律问题清单、避坑图文、流程说明、资料型笔记。

目标：让正在搜索问题的人通过图文进入私域。

矩阵搭建选稿拆解图文制作AB 链评论区链路标签优化
截

截流打法

定位：承接已经在看同行内容的高意向用户。

内容方向：同行笔记评论区、自营打窝笔记、关键词笔记。

目标：把已有需求用户引导到自己的资料钩子和私域。

找窝钩子文案库评论区话术截流素材库安全导粉
搜

电商搜索渠道

定位：用低门槛资料品承接有明确问题的用户。

内容方向：法律资料包、合同模板、维权流程、证据整理表、咨询前准备清单。

目标：先用资料完成首单信任，再引导进入私域咨询。

资料品设计标题关键词详情页逻辑客服话术加微路径
04 / 私域承接系统

加到微信不是终点，真正要搭的是承接、分层、培育和跟进系统。

法律服务的私域不能只做“发资料”。用户进来后要能被识别、被培育、被引导到咨询和服务。

01企微/个微基础建设

头像、昵称、朋友圈基础内容、欢迎语、风控和养号建议。

02欢迎语与资料承接

用户进来后知道能拿到什么、怎么描述问题、下一步找谁。

03用户标签与分层

按法律问题、紧急程度、付费意愿和用户价值做分层。

04朋友圈 SOP

持续输出专业内容、案例拆解、资料说明和咨询价值。

051v1 跟进 SOP

根据用户状态匹配不同话术和跟进频率。

06社群资料培育

用资料、案例和活动承接泛需求用户，筛出咨询用户。

07咨询转化与售后

完成首单、咨询、律师服务承接，并沉淀反馈和案例。

05 / 社群、转化与售后

资料社群负责培育，1v1 负责转化，售后负责沉淀信任素材。

社群不是闲聊群，而是资料培育群、需求筛选群、咨询转化群。

1

建群破冰

明确这个群能获得什么，降低用户防备，让资料粉愿意留下来。

2

资料分享

持续输出法律资料、模板、清单，让用户形成持续关注。

3

案例拆解

用真实场景讲清楚问题，让用户知道自己处在哪种情况。

4

咨询引导

把高意向用户从社群引到 1v1 或付费咨询。

5

晒单种草

展示咨询反馈、资料使用反馈、服务过程反馈，增强信任。

6

洗粉分层

把纯资料用户、咨询用户、高价值用户分开，不同人走不同路径。

7

裂变转介绍

通过资料包、专题群、老用户推荐继续扩散新用户。

引流品

资料包 / 模板 / 流程清单

用于承接公域搜索和私信用户，完成低门槛信任建立。

→

让用户把具体问题说出来，进入专业判断环节。

→
转化品

→
升单品

企业法务顾问 / 长期法律服务

承接企业客户和长期需求，建立复购与转介绍空间。

06 / 裂变宣传与内容反哺

裂变不是简单拉人，核心是把资料、案例和服务反馈变成可传播素材。

法律行业更适合资料型裂变、场景型裂变和口碑型转介绍。用户有理由转，裂变才自然。

私域沉淀怎么回到宣传渠道

  • 用户常问问题转成小红书图文选题
  • 咨询中的高频痛点转成律师 IP 短视频口播
  • 资料领取数据反推哪些法律方向更容易获客
  • 成交案例改写成案例拆解内容
  • 社群互动问题做成问答型内容
  • 售后反馈/好评做成朋友圈、社群、图文信任素材

08 / 合作后详细规划

正式合作后，将进一步拆到执行表、责任人和具体话术。

当前页面用于讲清楚大逻辑。最终落地版本，需要在同频会议和资料诊断后继续细化。

01

同频会议

明确业务目标、产品结构、律师资源、现有客户来源、历史成交数据、团队配置。

02

现有链路诊断

检查当前账号、微信、社群、销售话术、资料包、朋友圈、成交路径有没有断点。

03

产品结构梳理

明确引流品、首转品、转化品、升单品，不能只做获客，不设计后端承接。

04

前端渠道规划

明确 IP 起号、小红书矩阵、截流、电商搜索分别承担什么角色。

05

私域 SOP 制作

输出欢迎语、标签体系、朋友圈 SOP、1v1 话术、社群 SOP、售后 SOP。

06

数据看板搭建

统计每个渠道的进粉、首单、咨询、成交、复购、转介绍数据。

07

阶段复盘优化

根据数据调整选题、资料钩子、承接话术、社群活动和销售路径。

提醒

最终方案不会停留在路径图

正式合作后会进一步拆解到执行表、时间节点、责任人、具体话术、内容选题和 SOP 文档。

闭环总结

这套路径的核心，是把前端内容、私域承接、销售转化、售后服务和内容宣传串成一条连续链路。

前端负责持续获取法律需求用户；私域负责承接、分层和培育；销售负责把需求转成咨询和服务；售后负责沉淀反馈、案例和转介绍；内容团队再把真实问题和案例反哺到 IP、图文矩阵、截流和电商搜索渠道。最终形成可持续迭代的法律服务增长闭环。
